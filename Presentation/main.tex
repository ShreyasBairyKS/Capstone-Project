\begin{frame}{Domain 1 \& 2: CNN-Based Defect Detection \& YOLO Object Detection}

\small
\begin{block}{Key Finding: Real-Time Multi-Defect Detection is Feasible}
Papers by Chatchapol et al.\ (ICCAS 2024), Kavitha \& Mamatha (IJISRT 2024, GCAT 2024), and Nguyen et al.\ (VJCS 2025) demonstrate:
\begin{itemize}\setlength{\itemsep}{3pt}
    \item \textbf{YOLOv5/v8 achieves >90\% accuracy} on real production lines for bottle caps, liquid level, and packaging defects
    \item \textbf{Single-pass detection} (one-stage networks) is preferred over two-stage methods for speed
    \item \textbf{Hardware integration proven:} IoT sensors + PLCs (S7-1200) + embedded systems (Jetson NX) enable end-to-end automation
\end{itemize}
\end{block}

\vspace{6pt}
\textbf{Highlighted papers:}
\begin{itemize}\setlength{\itemsep}{2pt}
    \item Chatchapol et al.\ — CNN for liquid fill (98.89\% accuracy) + conveyor speed control
    \item Nguyen \& Mac — YOLOv8 + multi-threading for real-time classification; scratch defects hardest to detect
    \item Kavitha et al.\ — YOLOv5 + Arduino actuation for multi-material sorting (cloth, PVC, PCB)
\end{itemize}

\end{frame}

% ============================================================
% SLIDE 5: Domain 3 & 4 --- Food Quality \& Industrial Vision
% ============================================================
\begin{frame}{Domain 3 \& 4: Food Quality Assessment \& Industrial Vision Systems}

\small
\begin{block}{Key Finding: Domain-Specific and Comprehensive Multi-Parameter Inspection}
Farhangi et al.\ (CCDS 2024), Sanjay (ICCSP 2024), and Du (MMIA 2025) show:
\begin{itemize}\setlength{\itemsep}{3pt}
    \item \textbf{Multi-parameter inspection} (fill level + cap + label + surface) achievable in single pass: \textbf{95.6\% average accuracy}
    \item \textbf{Food-specific defects} (oil quality, contamination) require specialized algorithms; classical vision + deep learning are complementary
    \item \textbf{Online (real-time) inspection outperforms offline:} 98\% accuracy vs.\ 85--92\%, 85\% faster processing (3 sec vs.\ 20 sec per unit)
\end{itemize}
\end{block}

\vspace{6pt}
\textbf{Highlighted papers:}
\begin{itemize}\setlength{\itemsep}{2pt}
    \item Farhangi et al.\ — LabVIEW + classical vision; 100\% cap accuracy, 95\% label placement
    \item Sanjay — CNN for food oil quality; real-time cost-effective alternative to lab testing
    \item Du — Multi-sensor fusion + edge computing; 60\% efficiency gain, 4.85\% scrap reduction
\end{itemize}

\end{frame}

\end{itemize}
\end{block}

% SLIDE 13: MC Dropout Theory
% ============================================================
\begin{frame}{Uncertainty Quantification --- MC Dropout Theory}

\small
\textbf{Monte Carlo Dropout} estimates model \textbf{epistemic uncertainty} (model knowledge gaps) at inference time by keeping dropout layers active:

\vspace{8pt}
\begin{block}{MC Dropout Methodology}
\begin{enumerate}\setlength{\itemsep}{6pt}
    \item \textbf{Multiple forward passes}: run the classifier $n$ times on the same input (default: $n=20$)
    \item \textbf{Stochastic variation}: each pass applies different random dropout masks
    \item \textbf{Aggregate outputs}: collect all $n$ predicted confidences for the defect class
    \item \textbf{Compute statistics}: 
    \begin{itemize}
        \item Mean confidence: $\hat{\mu} = \frac{1}{n} \sum_{i=1}^{n} c_i$
        \item Standard deviation (uncertainty): $\hat{\sigma} = \sqrt{\frac{1}{n}\sum_{i=1}^{n}(c_i - \hat{\mu})^2}$
        \item 95\% confidence interval: $[\hat{\mu} - 1.96\hat{\sigma}, \hat{\mu} + 1.96\hat{\sigma}]$
    \end{itemize}
    \item \textbf{Flag uncertain predictions}: if $\hat{\sigma} >$ threshold (0.15), mark as \textit{uncertain}
\end{enumerate}
\end{block}

\end{frame}

% ============================================================
% SLIDE 14: MC Dropout in VisionFood
% ============================================================
\begin{frame}{MC Dropout Application in VisionFood QAI}

\small
\begin{block}{Integration with EfficientViT Classifier}
\begin{itemize}\setlength{\itemsep}{6pt}
    \item \textbf{Model export}: classifier exported to ONNX format \textit{with dropout enabled} (using \texttt{training=True} mode)
    \item \textbf{Inference flow}:
    \begin{enumerate}
        \item Extract product crop from detection bounding box
        \item Run ONNX session 20 times on the same crop image
        \item Collect all 20 confidence scores $\{c_1, c_2, \ldots, c_{20}\}$
        \item Compute mean, std, and confidence interval from samples
    \end{enumerate}
    \item \textbf{Uncertainty thresholds} (tunable via \texttt{EdgeConfig}):
    \begin{itemize}
        \item UQ threshold = 0.15: if $\hat{\sigma} > 0.15$, mark \textit{is\_uncertain} = True
        \item Escalation threshold = 0.60: if mean confidence $< 0.60$ \textit{AND} uncertain, escalate to fog tier
    \end{itemize}
    \item \textbf{Result stored in} \texttt{UQResult} schema with full statistics for explainability
\end{itemize}
\end{block}

\end{frame}

\begin{frame}{REMEDY Engine --- Architecture Overview}

\small
\textbf{REMEDY} (Rapid Evaluation, Mitigation, and Diagnosis of Yield Errors) routes defects to corrective actions:

\vspace{6pt}
\begin{figure}
    \centering
    \small
    \begin{tabular}{c|c|c}
        \textbf{Input} & \textbf{Component} & \textbf{Output} \\
        \hline
        Detection + UQ & \textit{SeverityScorer} & Severity Grade (S1--S4) \\
        \hline
        Grade + Class & \textit{TriageRouter} & Remediation Action \\
        \hline
        SKU ID & \textit{SKUProfileManager} & Config Overrides \\
    \end{tabular}
\end{figure}

\vspace{8pt}
\begin{block}{Severity Grades}
\begin{itemize}\setlength{\itemsep}{3pt}
    \item \textbf{S1 (Minor)}: remediable on-line at dedicated station (e.g., relabel, clean)
    \item \textbf{S2 (Moderate)}: requires intermediate handling (e.g., refill, repack)
    \item \textbf{S3 (Severe)}: rejected; does not re-enter production
    \item \textbf{S4 (Critical)}: quarantined; flagged for investigation
\end{itemize}
\end{block}

\end{frame}

% ============================================================
% SLIDE 16: Severity Scoring Formula
% ============================================================
\begin{frame}{REMEDY Severity Scorer --- Weighted Formula}

\small
\textbf{Severity score} (range [0, 1]) is computed as a weighted sum of four risk components:

\vspace{1pt}
\begin{equation*}
\text{score} = w_{\text{area}} \cdot a + w_{\text{conf\_uq}} \cdot u + w_{\text{class}} \cdot r + w_{\text{attempt}} \cdot p
\end{equation*}

\vspace{1pt}
\begin{block}{Component Definitions}
\small
\begin{itemize}\setlength{\itemsep}{4pt}
    \item $a$ = \textbf{Area ratio}: defect bounding box area / total image area (capped at 0.25)
    \item $u$ = \textbf{Uncertainty component}: MC Dropout std dev $\hat{\sigma}$ (capped at 0.30)
    \item $r$ = \textbf{Class risk}: food safety impact of defect class (e.g., contamination=1.0, misalignment=0.30)
    \item $p$ = \textbf{Attempt penalty}: prior remediation failures (capped at 1.0, increases by 0.5 per attempt)
\end{itemize}
\end{block}

\vspace{1pt}
\begin{block}{Weights (from \texttt{EdgeConfig})}
$w_{\text{area}} = 0.35$, \quad $w_{\text{conf\_uq}} = 0.15$, \quad $w_{\text{class}} = 0.40$, \quad $w_{\text{attempt}} = 0.10$
\end{block}

\vspace{1pt}
\begin{block}{Grade Assignment}
\small
\textbf{Thresholds}: S1 if score $< 0.30$; S2 if $< 0.55$; S3 if $< 0.80$; S4 if $\geq 0.80$
\end{block}

\end{frame}

% ============================================================
% SLIDE 18: Remediation Actions
% ============================================================
\begin{frame}{Remediation Actions --- Corrective Measure Details}

\small
\begin{block}{Action Definitions and Station Assignments}
\begin{itemize}\setlength{\itemsep}{5pt}
    \item \textbf{RELABEL}: re-apply or correct label (Station A)
    \item \textbf{REFILL}: adjust product level or contents (Station B)
    \item \textbf{REPACK}: replace damaged packaging (Station C)
    \item \textbf{CLEAN}: surface decontamination (Station C)
    \item \textbf{REJECT}: remove from line permanently (Quarantine)
    \item \textbf{PASS}: product proceeds to shipping (no intervention)
\end{itemize}
\end{block}

\vspace{8pt}
\small
Each remediation action is routed to its corresponding station, where operators perform the corrective measure. Severe defects (S3/S4) and products exceeding max remediation attempts are automatically rejected, bypassing all stations.

\end{frame}

\begin{frame}{Product Customization --- SKU Profiles}

\small
\textbf{SKU Profile Manager} loads per-product YAML configurations to override REMEDY parameters:

\vspace{8pt}
\begin{block}{YAML Configuration Structure}
\begin{itemize}\setlength{\itemsep}{5pt}
    \item \textbf{class\_risk\_overrides}: per-class risk weights (e.g., \texttt{contamination: 0.95}, \texttt{filling: 0.80})
    \item \textbf{rejection\_area\_thresholds}: maximum defect size before forced rejection per class
    \item \textbf{preferred\_stations}: hint for routing (e.g., bottle\_250ml prefers Station B for filling checks)
    \item \textbf{uq\_threshold\_override}: product-specific uncertainty escalation threshold
\end{itemize}
\end{block}

\vspace{8pt}
\begin{block}{Example: \texttt{bottle\_250ml.yaml}}
\tiny
\texttt{sku\_id: "bottle\_250ml"} \\
\texttt{class\_risk\_overrides:} \\
\texttt{\hspace*{0.2cm}improper\_filling: 0.85  \# Higher risk} \\
\texttt{\hspace*{0.2cm}label\_misalignment: 0.35} \\
\texttt{rejection\_area\_thresholds:} \\
\texttt{\hspace*{0.2cm}packaging\_damage: 0.10} \\
\texttt{preferred\_stations:} \\
\texttt{\hspace*{0.2cm}improper\_filling: "B"}
\end{block}
